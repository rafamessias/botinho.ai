## Company membership model

Firestore path: `companies/{companyId}/members/{uid}`

| Field | Type | Purpose |
|-------|------|---------|
| `isOwner` | boolean | Company owner |
| `isAdmin` | boolean | Admin privileges |
| `canPost` | boolean | Can create/send content |
| `canApprove` | boolean | Approval permission (reserved) |
| `status` | enum | `invited` \| `accepted` \| `rejected` |

Only members with `status: accepted` can access company data.

## resolveCompanyContext

File: [`components/server-actions/utils.ts`](../../components/server-actions/utils.ts)

Central guard used by most server actions.

```typescript
resolveCompanyContext({
  companyId?: string,      // defaults to user.defaultCompanyId or first membership
  requireAdmin?: boolean,  // requires isAdmin
  requireCanPost?: boolean // requires isAdmin OR canPost
})
```

### Resolution order for companyId

1. Explicit `companyId` argument
2. `users/{uid}.defaultCompanyId` from Firestore
3. First accepted membership via collection group query on `members`

### Errors thrown

| Error message | Cause |
|---------------|-------|
| `Not authenticated` | No session / uid |
| `Company not selected` | No company found |
| `Not authorized for this company` | No accepted membership |
| `Requires admin permissions` | `requireAdmin` failed |
| `Insufficient permissions` | `requireCanPost` failed |

Returns: `{ companyId, userId, membership }`.

## Permission usage by feature

| Feature | Guard |
|---------|-------|
| Most read actions | Default membership |
| Company settings update | `requireAdmin: true` |
| Member invite/remove | Admin checks in action |
| Company token generate/regenerate | Admin + plan API access check |
| Inbox send / create conversation | `requireCanPost: true` |
| AI training CRUD | `requireCanPost: true` |

## Company API token

Field: `Company.tokenApi` (nullable bcrypt hash)

Generated by `generateCompanyTokenAction`. Used for external API access (survey-era endpoints removed; token infrastructure retained).

## Webhook and cron auth

| Route | Auth mechanism |
|-------|----------------|
| `/api/stripe/webhook` | Stripe signature (`STRIPE_WEBHOOK_SECRET`) |
| `/api/auth/*` | NextAuth internal |

No messaging webhooks exist yet.

## Edge cases

- Company ids are Firestore string document ids (not integers).
- Collection group query on `members` requires Firestore index for `uid + status`.
- Page middleware only checks session cookie — authorization happens in server actions.
