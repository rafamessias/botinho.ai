## Supported locales

| Locale | Language | Default |
|--------|----------|---------|
| `en` | English | Yes |
| `pt-BR` | Brazilian Portuguese | No |

Configured in [i18n/routing.ts](../../i18n/routing.ts):

```typescript
locales: ['en', 'pt-BR']
defaultLocale: 'en'
```

## URL strategy

All localized pages use prefix: `/en/dashboard`, `/pt-BR/inbox`.

Middleware redirects unprefixed paths to `/{defaultLocale}{path}`.

## next-intl setup

- Plugin in [next.config.mjs](../../next.config.mjs): `createNextIntlPlugin('./i18n/request.ts')`
- Locale layout wraps children with messages: [app/[locale]/layout.tsx](../../app/[locale]/layout.tsx)
- Request config loads message JSON by locale: [i18n/request.ts](../../i18n/request.ts)

## Navigation helpers

[i18n/navigation.ts](../../i18n/navigation.ts) exports locale-aware:

- `Link`
- `redirect`
- `usePathname`
- `useRouter`

Use these instead of Next.js defaults for correct locale prefixing.

## Message catalogs

Large JSON files (~100KB each):

- [i18n/messages/en.json](../../i18n/messages/en.json)
- [i18n/messages/pt-BR.json](../../i18n/messages/pt-BR.json)

Namespaces include: Dashboard, Inbox, Customer, AiTraining, Company, Settings, Auth forms, Landing, Subscription, etc.

## Language switching

### UI

[components/language-selector.tsx](../../components/language-selector.tsx) — dropdown in site header.

### Persistence

1. **User profile** — `User.language` enum (`en` | `pt_BR`) via `updateUserLanguageAction`
2. **Cookie** — `user-language` cookie set in NextAuth JWT callback (readable by middleware)
3. **Middleware** — redirects authenticated users to preferred locale

### DB vs URL mapping

| DB (Language enum) | URL locale |
|--------------------|------------|
| `en` | `en` |
| `pt_BR` | `pt-BR` |

## UserProvider

Loads user language preference and syncs with next-intl on client.

## Email localization

Transactional emails use locale at send time for subject lines and template props (see [13-email-system.md](13-email-system.md)).

## Adding a new locale (documented procedure)

From readme:

1. Create `i18n/messages/{locale}.json`
2. Add locale to `i18n/routing.ts`
3. Update middleware matcher in `middleware.ts`

## Edge cases

- Prisma `Language` enum replaced by Firestore `UserLanguage` (`pt_BR` in DB; `pt-BR` in URLs).
- Sign-up OTP page reads `OTP_ENABLED` at server render time.
- Legacy Opineeo/survey i18n namespaces (`Survey`, `CreateSurvey`, etc.) remain unused — safe to prune in a future cleanup.
